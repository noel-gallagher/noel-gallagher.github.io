Two-phase commit

Two phase commit is a method of guarnteeing all participants of a distributed system perform a transaction. The action should not be performed until all memebers of a cluster have successfully sent acknowledgement.

The process is performed, unsurprisingly, in two phases: Prepare & Commit

In the prepare phase, the coordinator asks each node to PREPARE to perform the action. Each node then informs the coordinator if it has the capability to perform the transaction. If a node is unable to perform the action, it will send the coordinator an ABORT. The coordinator then tells all other nodes not to perform the action. Otherwise, the node will repspond with a READY response.

For the commit phase, the coordinator will send a COMMIT message to each member of the cluster. On receipt of this COMMIT message, each node will perform the transaction and send an acknowledgement. If there is an issue, the participant will send an ABORT message to the coordinator. The coordinator will then rely this ABORT message to the remaining clients. Each client will then each rollback the previous transaction. Otherwise, if all nodes have succesfully performed the action, the coordinator will send a COMMIT message to each member so that the transcation can be marked complete.
